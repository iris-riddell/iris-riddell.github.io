JavaScript, the DOM, and You

Well, I think we can all agree, that was quite a week! I packed so much JavaScript into my brain I’m surprised there isn’t object-oriented language oozing out my ears. JavaScript really is a world apart, when compared with HTML and CSS. If I were to use yet another analogy, and compare a website to the human body… I would say that HTML is the skeleton, CSS is the skin, eyes, hair and so forth, and JavaScript is the muscles. While HTML provides the structure and CSS makes everything look stylish, JavaScript makes everything go. Live feed updates? JavaScript. Form submission? JavaScript. Browser-based stop watch? You best believe that’s JavaScript.

Control Flow

The control flow is, simply, The Way In Which We Do Things. Or at least, it’s the way your computer does things. Code generally runs from top to bottom, unless it hits something that disrupts that flow. What would a control flow look like for something from everyday life? The DOM and how to interact with it

I confess, I really struggled to make sense of the DOM at first. It seemed like every Google search brought up a different result for what the DOM was, and what it did. If I could have live-Tweeted my thoughts during my research, it might have looked something like this:

“Okay, cool. It’s a representation model.”

“Uh, wait. It’s a standardised platform. So is it just a way of thinking about the structure of a web page?”

“Ooh. Oh. Nope, looks like you can interact with it. Through… JavaScript. Gotcha.”

“But… how?”

“Dev Tools? Okay, neat. I’ve definitely heard of that.”

“But… why?”

“Ah, you can manipulate elements so they can update page data without refresh? Make animations or drag, move, and delete elements? That’s pretty sweet.”


Once I got over my existential panic, I found the potential pretty exciting!

You can manipulate the DOM through Dev Tools. My exposure has been pretty limited so far, but one of the things you can do is pretend to be a hacker and superficially alter a bunch of websites, so I’m pretty excited about that. More to the point, you can actively alter elements on a page in real time, or even alter your code if you know what you’re doing.

Functions and why they are helpful

JavaScript is an object-based language and functions are one of those objects. As such, functions are a really crucial building block in JavaScript. These hardworking chunks of code basically exist to execute a task or calculate a value, before returning a result. They form the foundation of much of our code.
